Case Study Unit #4: The Cherry Blossom Race¶

By: Armando Vela & Jeff Washburn

Date: 9/22/2020

Abstract¶

Scraping web data, transforming the associated text, and creating a presentable dataframe was the topic of this case study. The data gathered and used was female ten-mile race data from the Cherry Blossom website. Techniques utilized throughout this case study include web scraping, function creation, regex pattern utilization, variable type modifications, and dataframe creation. Finally, we did not impute any 'NA' values as we left that up to the discretion of future researchers since it is outside the scope of the current assignment. With that said, the final dataframe is a great representation of the Cherry Blossom race data found within their website from 1999 through 2012.

Introduction¶

Chapter 2 of the 'Data Science in R: A Case Studies Approach to Computational Reasoning and Problem Solving' text book, discusses using R to review the Cherry Blossom Race. With that in mind, Python was used to address the case study question and scrape female runner web data and place within a clean dataframe for future analysis.

The specific race data used was from the Credit Union Cherry Blossom Ten Mile Run at Washington D.C. and is located at www.cherryblossom.org. The years gathered and transformed are female ten-mile races from 1999-2012.

The Methods section below walks through the steps taken from how we scraped the race data and the necessary formatting & cleaning steps taken to create the final 'All Races' dataframe.

Methods¶

Web Scraping¶

Data preparation for the Cherry Blossom race data required a variety of techniques to account for the different formats seen not only in a single file, but also the different formats across years.

Using Python, an API call was made to access each of the corresponding female race data’s HTML page. The extractResults function provided the parsing of the HTML race table data. We tried to be as generic as possible on parsing the race table data; however a couple of years did require special attention. For example, 2000, we had to look for a different identifier (face='Courier New') to pull the female’s race data placed after that specific tag.

Data Processing¶

Once the female’s race data ascertained, we then processed each year’s data. To process each year, we initially started with the headerCleaner and splitIdentifier methods.

  • The headerCleaner was used to remove unnecessary rows found at the beginning of the scraped data.
  • The splitIdentifier was used to identify the appropriate string splits to separate the scraped data accordingly. To put it simply the function looked at the starting position of the table columns based on an indicator character of the equals sign (“=”). Using this approach, we could then identify the appropriate string lengths per column for each racer. This approach seemed to work for all the race data except for 2001.

To handle the 2001 data, we took a different approach and used regex patterns to process the data. Using the grouping mechanism of regex allowed us to easily process the 2001 race data while simultaneously creating a dataframe.

See figures below comparing a Normal Process Example (ie. 'Indicator Method) and why the Special Regex method was needed.

Creation of Initial Dataframes by Year & Combining Dataframes by Year into one "AllRaces" Dataframe¶

Once we identified the necessary splits using the non-Regex method, using the dataSplitter function we created a single dataframe that represented each year of the female race data.

Once we had each year parsed, we then needed to perform some additional cleanup such as renaming columns or removing extra rows that did not provide any valuable racing information.

Note the renaming of columns was necessary so that each dataframe year had matching column names. In other words, some races simply had Time, while other used Gun Time. By renaming the columns to match across the data frames allowed us to combine into a single data frame much simpler.

It is important to point out that no imputation was done for NA values. This was chosen to allow the researcher to modify/impute the data as they see fit.

Conclusion¶

Processing the Cherry Blossom race data for the female runners did have its challenges as mentioned above. Switching over to regex patterns proved to be useful and moving forward on future race data seems like a good alternative to pursue as it seems more straightforward and intuitive on how to process the data. In addition, using regex patterns, there are no additional lines that are added to the dataframe, only the lines that match the pattern are added.

In conclusion, we feel that having all the data combined into a single data frame will be useful for future analysis. Note, we are aware that additional steps can be taken within the dataframe (ie. imputations, adjust data types, feature creation) to further improve the information displayed.

  • splitIdentifier(): Calculates start and end points to help separate text data.

def extractResults():
    """
    Name: extractResults function
    Purpose: Takes a list of websites from the cherry blossom race to download 1999-2012 race data for female participants.
    Parameters: NONE
    Packages/Functions Needed: Requires 'requests' and 'BeautifulSoup' libraries
    Return Output: dictionary listing year of race and corresponding race results
    """   
    
    #empty dictionary that will be filled in with year and associated race results.
    resultsDictionary = {}
    
    #empty list that will be filled in with associated race results
    resultsList = []
    
    for i in urls:
        yr = i[37:41]
        
        print(f'Working on {yr}!')
        
        #retrieve data from web site
        page = requests.get(i)        
        
        #perform initial html parsing
        soup = BeautifulSoup(page.content, 'html.parser')        
        
        if yr == '2000':
            #Year 2000 race does not have 'pre' tag.  Used a different identifier to locate text.
            results_info_with_tag = soup.find(face="Courier New")
        else:
            #identify race results with tag
            results_info_with_tag = soup.find('pre')

        #gather race data
        race_text = results_info_with_tag.get_text()

        #split string into different lines
        race_data_list = race_text.splitlines()

        #append race_data_list to resultsList
        resultsList.append(race_data_list)
        

    for idx in range(len(resultsList)):
        #create year variable to include as the dictionary key. Converted to string for dictionary purposes.
        year = 1999 + idx
        year = str(year)
    
        #combine year and results list into one dictionary
        resultsDictionary[year] = resultsList[idx]
        
    #return dictionary with year and race results
    return(resultsDictionary)

def headerCleaner(data): 
    """
    Name: headerCleaner function
    Purpose: Removes unnecessary header information from race data.
    Parameters: race results data
    Packages/Functions Needed: NONE
    Return Output: dictionary listing year of race and corresponding race results
    """       
    cleanDictionary = {}
    raceData = data
    
    for idx in range(len(raceData)):
        #create year variable to include as the dictionary key. Converted to string for dictionary purposes.
        year = 1999 + idx
        year = str(year)
        race = raceData[year]
        char = 0
        
        if year == '2001':
            cleanDictionary[year] = race[3:]
        else:
            for j in range(len(race)):

                if race[j] == '':
                    pass
                elif race[j][0].isalpha() and char ==0:
                    #combine year and results list into one dictionary
                    cleanDictionary[year] = race[j:]
                    char = 1
                
        
    #creturn clean dictionary
    return(cleanDictionary)

def splitIdentifier(data):
    """
    Name: splitIdentifier function
    Purpose: Takes race data, idenfities location of '=' sign, provides the location where there is a space between each '='
    Parameters: race results data
    Packages/Functions Needed: Requires 'pandas' library
    Return Output: dictionary listing year of race and corresponding index where a space was found between text columns
    """  
    raceData = data
    indexerDf = pd.DataFrame(columns=('year','idx','count'))
    years = list(raceData.keys())

    #iterate over list of years and gather row index where '=' are listed
    for i in years:
        tempData = raceData[i]

        for k in range(len(raceData[i])):
            count = raceData[i][k].count('=')

            if count > 0:
                indexerDf = indexerDf.append({'year': i ,'idx' : k,'count' : count}, ignore_index=True)

                
                
    #get dictionary of all breaks in (ie. spaces) between columns to help separate the race results information
    line_idx_dictionary = {}

    for row in range(len(indexerDf)):
        row_year = indexerDf.iloc[row,0]
        row_idx = indexerDf.iloc[row,1]
        line_idx_list = []

        for x in range(len(raceData[row_year][row_idx])):
            line_idx = raceData[row_year][row_idx].find(' ', x)

            if line_idx == x:
                line_idx_list.append(line_idx)  

        line_idx_dictionary[row_year] = line_idx_list   
    
    return(line_idx_dictionary)

def dataSplitter(data, year):
    """
    Name: dataSplitter function
    Purpose: Takes race data, uses splitIdentifier() function to split columns and place within a returned dataframe
    Parameters: data, year
    Packages/Functions Needed: Requires 'pandas' library
    Return Output: dataframe of race data
    """      
    raceData = data[str(year)]
    
    raceResultsDictionary = {}
    #raceResultsList = []
    
    splitID = splitIdentifier(data)
    splitsByYr = splitID[str(year)]
    
    df = pd.DataFrame()
    
    if year == '2003':
        adjuster_if_2003 = 1
    else:
        adjuster_if_2003 = 0
        
    
    for x in range(len(raceData)):
        
        if x==0:
            titleList = []
            
            for s in range(len(splitsByYr)):
                if s==0:
                    titleList.append(raceData[x][0:splitsByYr[s] + adjuster_if_2003])
                else:
                    titleList.append(raceData[x][splitsByYr[s-1] + adjuster_if_2003:splitsByYr[s] + adjuster_if_2003])
            
        elif x==1 and raceData[x].startswith('='):
            pass
            
        else:
            
            raceResultsList = []
            
            for s in range(len(splitsByYr)):
                if s==0:
                    raceResultsList.append(raceData[x][0:splitsByYr[s] + adjuster_if_2003])
                else:
                    raceResultsList.append(raceData[x][splitsByYr[s-1] + adjuster_if_2003:splitsByYr[s] + adjuster_if_2003])
            
            df = df.append(pd.DataFrame(raceResultsList).transpose(), ignore_index=True)           

    df.columns = titleList
        
    return(df)

def regexSplitter(data, year):
    """
    Name: regexSplitter function
    Purpose: Takes race data, uses RegEx patter to split into columns and return dataframe
    Parameters: data, year
    Packages/Functions Needed: Requires 'pandas' library, and 're' library
    Return Output: dataframe of race data
    """  
    raceData = data[str(year)]
    
    pattern2001 = "\\s+(\\d+)\\s+(\\d+)\\s+((?:[A-Za-z\\-\\.\\'\\` ]{1,25} ?\\b){1,4})\\s+(\\d+)\\s+((?:[A-Za-z\\-\\.\\'\\` ]{1,25} ?\\b){1,4})\\s+((?:(?:\\d+:){1,3}(?:\\d+)))\\s+((?:(?:\\d+:){1,3}(?:\\d+)))"
    
    df = pd.DataFrame()

    
    for x in range(len(raceData) - 1):

        row = list(re.findall(pattern2001, raceData[x])[0])
        
        df = df.append(pd.DataFrame(row).transpose(), ignore_index=True)

    
    df.columns = ['Place', 'Num', 'Name', 'Age', 'Hometown', 'NetTime', 'GunTime']

    
    return (df)
